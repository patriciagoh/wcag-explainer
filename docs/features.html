WCAG Explainer · The App

From reference to workflow tool

The explainer started as a browsable list of 87 criteria. These eight features turn it into something a team actually works in — connecting the scanner you run, the code you write, the users you serve, and the checklist you ship.

Find & fix

Start from what your tools flag

Developers see scanner output, not criterion numbers. These features bridge "my linter yelled at me" to "here's the criterion and the fix."

Feature 1 & 7

Rule lookup — axe-core & eslint-plugin-jsx-a11y

Paste a rule id your scanner or linter reported and jump straight to the criteria it maps to, each with the plain-English why and the fix. Covers axe-core rules (from the dataset) and 31 eslint-plugin-jsx-a11y rules (curated mapping).

In the app: the Rule lookup tab.

Feature 5

Fix-it diff & copy

Every criterion's pass and fail examples can be flipped into a line-level fail → pass diff, so you see the minimal change — not two separate blocks. One click copies the example, or the fixed code.

In the app: the "Show fail → pass diff" toggle and Copy buttons on any criterion.

Understand

See it from the user's side

The spec tells you the rule. This tells you what actually happens to a person when you get it wrong — the thing no Understanding doc or "just ask an LLM" reliably gives you.

Feature 4

"What the user experiences"

Each of the 87 criteria carries a concrete broken vs. done-right pair, framed around the person most affected — a screen-reader user hearing a filename, a low-vision user losing text at 200% zoom, a keyboard user trapped in a widget.

In the app: the "What the user experiences" panel on every criterion.

Component & role filtering

Feature 8

Spot-the-violation quiz

Turns the existing pass/fail examples into flashcards: see a snippet, guess pass or fail, then get the answer plus a link to the criterion — with a running score. Built for onboarding a team to a11y.

In the app: the Quiz tab.

Feature 9

Project checklist & PR template

Add the criteria relevant to your project, name it, and export a Markdown checklist or a PR-review template. Your selection is saved in the browser, so it persists across visits.

In the app: the My checklist tab + "＋ Add to checklist" on any criterion.

Feature 10

Deep links & exports

Every criterion has a stable permalink (#1.4.3) you can copy and paste into Slack or docs, plus a one-click Copy as Markdown for issues and wikis.

In the app: the 🔗 and ⧉ buttons in each criterion's header.

Under the hood

How the new data is composed

Three features needed data that didn't exist (screen-reader experience, component/role tags, eslint mappings). Rather than touch the validated build-pipeline dataset, they live as side files composed onto each criterion at load — keeping the wcag-criteria.json contract clean.

These features layer on top of the Phase 2 enriched dataset without changing its contract — new data lives in side files composed by src/augment.ts. Built into the skill's template/, type-checked, and covered by the app test suite.
